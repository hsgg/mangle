Output format

Mangle can write files in any of the input formats:

  • c - circle
  • v - vertices
  • e - edges
  • r - rectangle
  • p - polygon
  • s - spolygon
  • R - Region
  • H - healpix_weight (rasterize only)

Additionally, there are six formats that can be written but not read:

  • a - area
  • g - graphics
  • i - id
  • l - list
  • m - midpoint
  • w - weight
    •

The output format of polygon files is compatible where possible with the input format. Output files created in circle, edges, polygon, rectangle, Region, spolygon, vertices, and healpix_weight formats can all be read as input polygon files. Output files created in area, graphics, id, midpoint, and weight formats cannot be read as input polygon files, because they contain too little information, or in the case of graphics and list formats because the interpretation on input would be ambiguous.

The default output format is polygon. The output format can be changed with the -o (output) switch.

(Note: Outputting healpix_weight format from rasterize uses the the -H switch instead.)

The only formats that retain complete information about polygons are the circle, polygon, and spolygon formats. The edges, rectangle, and vertices formats in general lose information about polygons.

Formats valid for both input and output
-oc circle  
-oe[<m>] edges <m> is an integer specifying the number of points per edge. The default is 2 points per edge, consisting of one vertex and one midpoint per edge.
[-op] polygon This is the default output format. The -op switch is permitted, but not required.
-or rectangle  
-oR Region  
-os spolygon A stripped down version of the polygon format.
-ov vertices  
-H healpix_weight Only rasterize can write files in healpix_weight format, so it uses a different switch (-H) rather than the generic -o switch.
Formats valid only for output
-oa area  
-og[<m>] graphics <m> is an integer specifying the number of points per (2 pi) of azimuthal angle around each edge. The default is 2 points per (2 pi). You will probably want to specify a larger number.
-ol[<m>] list <m> is an integer specifying the number of points per (2 pi) of azimuthal angle around each edge. The default is 2 points per (2 pi). You will probably want to specify a larger number.
-oi id  
-om midpoint  
-ow weight  
Output angular unit
-o<format>[..]<unit> circle, edges, rectangle, vertices, midpoint The output angular unit <unit> may be one of:
r radians, d degrees, m arcmin, s arcsec, h hms(RA) & dms(Dec).

The default output angular unit is d degrees.

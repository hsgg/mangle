list format

The list format is an adaptation of the graphics format designed for use with the MATLAB Mapping Toolbox, and may be more convenient to use with other plotting packages as well.

As in the graphics format, there is a specified number of points per (2 pi) of azimuthal angle along each edge. The azimuth and elevation for each point is printed on a separate line, and the points for each polygon (or rather, each connected part of each polygon) are separated by a line containing two NaNs. This produces a list of NaN-separated list of azimuth (longitude) and elevation (latitude) points, a format used by the MATLAB Mapping Toolbox and other terrestrial mapmaking tools.

In the list format, the weights of the polygons are stored in a supplementary file with one weight on each line for each connected polygon part in the NaN-separated list. This file is automatically named as polygon_outfile.weight, e.g., the command

poly2poly -ol30 mypolygons.pol mypolygons.list

produces the file mypolygons.list, a NaN-separated list of azimuths and elevations, and the file mypolygons.list.weight containing the corresponding weights.

The list format is implemented only as output, not as input, because of ambiguity in the interpretation of the format. That is, mangle can write a polygon file in list format, but it will refuse to read such a file.

On output, list format with m points per (2 pi) along each edge is specified with

-ol<m>

The default is m = 2, giving 2 points per (2 pi) of azimuthal angle, which is a pretty miserable resolution. In practice you will probably want higher resolution, with m = 12 or more.

A MATLAB script for plotting polygon files in list format is provided on the scripts page.
